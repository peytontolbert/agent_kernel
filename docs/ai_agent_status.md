## Honest Capability Verdict

- The kernel is no longer limited to compare-only bookkeeping on eligible non-protected lanes: supervised promotion can now perform governed finalize, stage one-round canary observation, trigger rollback on trust regression, and require post-rollback validation before promotion resumes.
- Protected and bootstrap-sensitive lanes still keep stricter review and rollout gates, so worker discipline remains `--generate-only` unless you are explicitly coordinating promotion.
- The remaining gap to robust coding agency is not finalize plumbing. It is unattended trust depth and breadth: broader counted evidence on `repository`, `project`, and `integration`, repeated walk-away proof on real workflows, and stronger evidence that rollback/verification catch hidden side effects.
- The coding stack is still not meaningfully syntax-aware in the live software loop. The repo has strong bounded preview and structured-edit machinery, but the runtime mostly reasons over retained text windows, commands, verifier results, and workflow state rather than parsing source into AST/CST structures for symbol-aware edits or semantic edit validation.
- That distinction should stay sharp: syntax-aware tooling is not the same thing as real software understanding, and humans do not program by manually executing AST rewrites either. But for a digital kernel it is still a meaningful missing motor layer, because import-safe edits, signature propagation, symbol-region targeting, and pre-execution syntax checks can be done more directly and more cheaply by machine structure than by repeated text traversal alone.
- The current repo-level AST use is mostly diagnostic, not runtime-governing: [`scripts/report_prompt_diagnostics.py`](/data/agentkernel/scripts/report_prompt_diagnostics.py) parses Python for prompt/payload inspection, while the live coding path still routes through retained preview windows and structured edit proposals in [`agent_kernel/modeling/policy/decoder.py`](/data/agentkernel/agent_kernel/modeling/policy/decoder.py).
